## 🎯 部署步骤概览

### 步骤 1：注册 Render（5 分钟）
- 访问 https://render.com
- 使用 GitHub 账号登录
- 创建新的 Web Service：timeline-app
- 连接 GitHub 仓库：californiafinch/timeline-app

### 步骤 2：配置环境变量（5 分钟）
- 在 Render Dashboard 中添加 3 个环境变量：
  - SUPABASE_URL
  - SUPABASE_KEY
  - JWT_SECRET
  - NODE_ENV

### 步骤 3：配置自定义域名（5 分钟）
- 在 Render Dashboard 中添加自定义域名
- 或使用默认域名：timeline-app.onrender.com

### 步骤 4：部署服务（自动）
- Render 会自动检测 GitHub 仓库更新
- 自动部署到 Render 服务器

### 步骤 5：配置 GitHub Pages（10 分钟）
- 在 GitHub 仓库中启用 GitHub Pages
- 配置源目录为根目录（排除 server.js）

### 步骤 6：更新前端代码（可选，1-2 小时）
- 修改 timeline-optimized.js 中的 API 基础 URL
- 从 `http://localhost:3000/api` 改为 `https://timeline-app.onrender.com/api`

### 步骤 7：测试和验证（30 分钟）
- 测试 GitHub Pages 静态文件
- 测试 Render API 端点
- 验证所有功能正常工作
